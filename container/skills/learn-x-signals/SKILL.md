# Learn from X Signals

Use X to discover how real people describe problems, trends, needs, complaints, and new workflows.

This skill is most useful for:

- finding emerging topics before SEO tools fully reflect them
- extracting real user language for keyword variants
- spotting repeated pain points or unmet needs
- validating whether a theme has active discussion behind it

## Core usage rules

1. Start with short search terms
   - Default to 1-2 word searches first.
   - Do not start with long sentence-like queries unless the user explicitly asks for that.

2. Refine only after signal appears
   - If the first search returns something relevant, then narrow with modifiers.
   - If the first search is too narrow and returns nothing useful, broaden instead of forcing it.

3. Look for language, not just popularity
   - Capture how users phrase the problem.
   - Look for repeated modifiers, product terms, verbs, frustrations, and desired outcomes.

4. Use X as a signal source, not final proof
   - X can reveal that a problem is real or rising.
   - It does not by itself prove SEO potential or ranking ease.
   - Pass promising candidates into `learn-google-trends` or `learn-serp-inspection` next.

## What to look for

- repeated pain points
- people asking for tools, templates, benchmarks, checkers, generators, or comparisons
- new terminology or phrasing that may become keywords
- fast-rising discussions with concrete use cases
- complaints about current products being bad, missing, expensive, or hard to use

## Output expectations

When reporting back, try to include:

- seed topic
- recurring phrases or keyword variants
- what users seem to want
- whether the signal looks weak, medium, or strong
- what should be checked next in Trends or the SERP

## Avoid

- using long over-specified queries too early
- assuming every viral topic is good for SEO
- confusing engagement with monetizable intent
- stopping at tweet summaries without extracting keyword variants
